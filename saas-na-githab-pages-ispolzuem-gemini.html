SaaS на Githab Pages, используем Gemini

Максим Прохоров 11/03/2026

Задача стоит не простая. Html + js = SaaS. 

Решаем:

Создание Doc-Maker.site (сейчас лежит на Github https://titomaxxx90.github.io/doc-maker/) — это отличный пример того, как современный «стек» технологий позволяет одному разработчику создать полноценный SaaS-продукт, который выглядит как работа целой команды.

Вот «подкапотная» выжимка того, как создавался этот проект, и какие инструменты легли в его основу:

1. Архитектура: «Frontend-only» с облачным бэкендом

Основная идея заключалась в том, чтобы сделать сервис максимально легким и быстрым. Вместо того чтобы поднимать сложный сервер с Node.js или Python, вся логика вынесена в браузер (Frontend).

  • HTML5 + CSS3 (Tailwind CSS): Это фундамент. Tailwind позволил собирать адаптивный интерфейс прямо в HTML-файлах, не отвлекаясь на написание сотен строк в .css файлах. Это в разы ускорило верстку блоков и карточек.

  • JavaScript (Vanilla JS): Весь функционал (калькулятор сумм, генерация текста прописью, управление состоянием приложения) написан на чистом JS. Это дает максимальную производительность без лишних библиотек.

2. «Мозг» и Хранение: Supabase

Это самый важный инструмент, который заменил собой написание громоздкого бэкенда.

  • Supabase (альтернатива Firebase) предоставил всё необходимое «из коробки»:

    • База данных (PostgreSQL): Хранит всех пользователей, историю их счетов и актов.

    • Auth API: Позволило реализовать регистрацию/вход за несколько строк кода, не задумываясь о том, как безопасно хранить пароли (всё делает Supabase).

    • RLS (Row Level Security): Мощный механизм безопасности, который гарантирует, что пользователь «А» никогда не увидит документы пользователя «Б», даже если он знает ID документа.

3. Генерация PDF: html2pdf.js

Это «магия», которая превращает ваш экранный макет в полноценный документ:

  • Библиотека html2pdf.js берет кусок HTML (вашу форму счета) и конвертирует его в PDF, сохраняя все шрифты, отступы и таблицы.

  • Она работает в связке с html2canvas, который «фотографирует» ваш div с документом перед конвертацией.

4. Инструменты разработки (Toolchain)

Процесс создания выглядел как итеративный цикл:

  1. Прототипирование: Сначала набрасывался «скелет» на обычном HTML.

  2. Итерации: Использование локального сервера для мгновенного отображения изменений (Live Server в VS Code).

  3. Деплой: Поскольку это статический сайт, его можно запустить где угодно. Для таких проектов идеально подходят Vercel или GitHub Pages.

  4. СУБД-менеджмент: Весь SQL-код для таблиц (ALTER TABLE, CREATE POLICY) тестировался прямо в SQL-редакторе Supabase, который встроен в их админ-панель.

Почему этот подход эффективен?

  • Скорость: Вы не тратите время на настройку серверов, Docker-контейнеров или API-эндпоинтов.

  • Масштабируемость: База данных PostgreSQL готова к огромным объемам данных.

  • Стоимость: Использование «Serverless»-решений (как Supabase) позволяет проекту работать бесплатно или за копейки на старте.

Проект создан по принципу «Минимально жизнеспособного продукта» (MVP): сначала логика, потом функционал, и в самом конце — упаковка в облако. Это позволяет очень быстро добавлять новые функции (как мы сейчас делаем с АВР), просто дописывая нужные функции в JS и обновляя схему БД.
